Go FlightRecorder 和 trace 使用指南

本文将介绍如何使用 Go 的 runtime/trace 包和 FlightRecorder 功能来捕获程序运行时的详细信息，并演示如何查看和分析生成的 trace 文件。

一、什么是 Go trace

Go trace 是 Go runtime 提供的一种机制，用于记录程序运行时的事件，例如：

  • Goroutine 创建、调度、阻塞和结束
  • 系统调用和网络 I/O
  • 垃圾回收事件
  • 用户自定义的任务和区域

使用 trace 可以帮助开发者分析程序性能瓶颈、定位死锁、阻塞或 GC 停顿问题。

二、FlightRecorder

FlightRecorder 是 Go 1.25 引入的新特性，类似程序的“黑匣子”，它可以在程序运行时持续记录 trace 数据，并维护一个滑动窗口，只保存最近的一段 trace。

四、查看和分析 trace

1. 打开 trace

浏览器会打开一个页面，提供丰富的可视化界面。

2. 常用视图

  • By Proc：按逻辑处理器查看 goroutine 执行情况
  • By Thread：按操作系统线程查看迁移情况
  • Goroutine analysis：按 main 函数聚合，查看阻塞、运行和调度情况
  • Profiles：包括网络阻塞、同步阻塞、系统调用、调度延迟
  • User-defined tasks/regions：显示你标注的代码任务，方便对齐业务逻辑
  • GC / Heap / Threads：观察垃圾回收和内存变化

3. 典型用途

  • CPU 占用高 → 查看 By Proc 中哪个 goroutine 占用时间长
  • 阻塞或死锁 → 查看 Goroutine analysis 和同步阻塞 profile
  • GC 停顿 → 查看 GC 时间线和 Minimum Mutator Utilization
  • 网络或文件 I/O 延迟 → Network / Syscall profile
  • 任务耗时分析 → User-defined regions

五、总结

  • FlightRecorder 提供了实时、滑动窗口的 trace 收集能力，适合持续运行的服务
  • WriteTo 可以随时导出 snapshot，生成 trace.out 文件
  • go tool trace 提供了丰富的可视化分析功能，能同时分析 CPU、I/O、GC、goroutine 等多维度指标
  • 使用 trace.WithRegion 或 trace.Log 可以标记关键任务或阶段，方便业务逻辑对齐分析
