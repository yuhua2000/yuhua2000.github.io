八、总结

WaitGroup.Go 的设计体现了 Go 并发库的一个重要原则：

不要让同步原语掩盖程序错误。

因此在 panic 场景下：

情况 行为
正常执行 Done
Goexit Done
panic 重新 panic

这样可以保证：

  • panic 不会被吞掉
  • 程序不会进入不一致状态
  • 并发语义保持正确
